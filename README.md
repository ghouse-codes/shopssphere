## 📌 Project Overview

ShopsSphere is a responsive e-commerce frontend application built with React. It solves a common real-world problem — giving users a clean, fast, and intuitive interface to browse and shop for tech products on both mobile and desktop devices.

This project was built to gain hands-on experience with React component architecture, state management, routing, and building production-ready UI for e-commerce use cases. It targets developers learning React and anyone who wants a solid e-commerce UI starter.

---

## ✨ Features

**🛒 Shopping Experience**
- Product listing with category browsing
- Product detail pages with descriptions
- Add to cart functionality
- Responsive mobile-first design

**🎨 UI & Design**
- Clean, modern interface built with CSS
- Mobile-optimized layouts
- Fast page loads with Vite build tool
- SEO-friendly pages using `react-helmet-async`

**⚡ Performance**
- Vite-powered lightning-fast dev server
- Optimized asset bundling
- Component-based architecture for reusability

## 🚀 Installation & Setup

### Prerequisites
- Node.js v18+
- npm v9+

### Steps

```bash
# 1. Clone the repository
git clone https://github.com/ghouse-codes/shopssphere.git

# 2. Navigate into the project
cd shopssphere/frontend

# 3. Install dependencies
npm install

# 4. Start the development server
npm run dev
```

Open your browser at `http://localhost:5173`

## 🧠 What I Learned

Building ShopsSphere pushed me to grow across multiple areas of frontend development:

- **React** — Understood component lifecycle, props, state, and hooks like `useState` and `useEffect` in a real project context
- **Component Design** — Learned how to break down a UI into small, reusable, and maintainable components
- **React Router** — Handled client-side navigation and dynamic routes for product pages
- **CSS Layout** — Practiced Flexbox and Grid to build fully responsive layouts from scratch
- **SEO with React** — Used `react-helmet-async` to manage document head tags for better SEO
- **Vite** — Set up a modern, fast development environment instead of the traditional Create React App
- **Git & GitHub** — Managed version control, resolved merge conflicts, and pushed a real project to a remote repository
- **Project Structure** — Organized files and folders following real-world frontend conventions
- **Debugging** — Traced component re-renders, fixed layout bugs across screen sizes, and resolved import errors
- **Clean Code** — Wrote readable, well-organized code with consistent naming conventions

## 💡 Why I Built This

I wanted to move beyond tutorial projects and build something that mirrors a real product. E-commerce is one of the most widely used application types in the industry, so building ShopsSphere gave me exposure to product listing logic, routing patterns, and mobile-first design — skills that directly translate to professional frontend work.

This project helped me understand how real React applications are structured and gave me confidence to work on production-grade codebases.
